<objective>
Stand up the thinnest working end-to-end ingest spine: a reel URL submitted via CLI becomes a `submission` + `job` row, a single worker loop claims the job from a durable SQLite-backed queue, downloads the reel with `yt-dlp --cookies-from-browser` (reusing the validated `scripts/fetch-reel.ts` approach), extracts audio + keyframes with ffmpeg, parses metadata from the `.info.json` sidecar, and persists a `findings` row pointing at the local media files.

This plan also re-platforms the repo off Cloudflare/Wrangler onto a plain Bun app (per D-01) so every later slice builds on a clean local foundation. It is the MVP vertical slice — after it, the user can run one command and get a stored Finding from a reel link.

Purpose: Deliver the load-bearing spine (URL -> queue -> worker -> yt-dlp+ffmpeg -> stored Finding) that all other capture paths (drop-folder, HTTP, saved-sync) plug into. Honors D-01..D-03, D-05..D-07, D-09, D-11..D-17.
Output: A runnable worker + CLI; `bun:sqlite` schema with submissions/jobs/findings; queue + download + media + metadata modules; tests including a failing-first end-to-end spine test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user/CLI -> yt-dlp/ffmpeg subprocess | user-supplied URL and output paths cross into subprocess arguments |
| Instagram (remote) -> .info.json parser | untrusted/attacker-influenced JSON crosses into the metadata parser |
| filesystem (yt-dlp output template / shortcode) -> MEDIA_DIR writes | externally-influenced filenames cross into local path joins |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | yt-dlp/ffmpeg invocation (download.ts, media.ts) | mitigate | All subprocesses via `Bun.spawn([...])` arg ARRAYS, never shell strings (Task 2 acceptance grep gate) |
| T-01-02 | Tampering | URL intake (cli.ts) | mitigate | Validate via `parseReelShortcode` — reject non-instagram-reel URLs before they reach yt-dlp (Task 3, SSRF/arbitrary-URL) |
| T-01-03 | Tampering | MEDIA_DIR writes (download.ts/media.ts) | mitigate | Confine output under MEDIA_DIR; `--restrict-filenames` + shortcode-keyed paths; reject path components resolving outside MEDIA_DIR |
