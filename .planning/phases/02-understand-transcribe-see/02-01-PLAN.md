<objective>
Wire the local enrichment pass end-to-end as the smallest sound extension of the existing SQLite job queue (D-01). After Phase 1 ingest persists a finding (`enrich_status='pending'`), the worker automatically enqueues and drains an **enrich** job that loads the finding's local audio/keyframes, runs transcription and vision concurrently, writes the results onto the finding, and flips `enrich_status` to `done` (or `failed` with the error surfaced).

This plan delivers the full vertical path with the two model calls STUBBED behind an `ENRICH_FAKE=1` seam (mirroring Phase 1's `INGEST_FAKE`), so the handoff, status transitions, idempotency, and column writes are proven in-sandbox without live API keys. Plans 02 (transcription) and 03 (vision) replace the stub bodies with real Groq/Claude calls.

Purpose: Establish the ingest→enrich handoff and the orchestrator contract so the real model calls drop into a working, tested pipeline.
Output: Extended queue/loop/schema, the `src/enrich/index.ts` orchestrator, stub `transcribe.ts`/`vision.ts` (real impls land in 02/03), and a passing end-to-end enrich test.
</objective>
