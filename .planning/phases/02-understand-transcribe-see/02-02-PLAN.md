<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local audio file → Groq API | audio bytes leave the host to a third-party transcription service |
| env → outbound header | GROQ_API_KEY placed in the Authorization header |
| Groq response → finding row | transcript text is attacker-influenceable (reel audio); stored as data only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-SEC2 | Information Disclosure | GROQ_API_KEY | mitigate | Key only in Authorization header; never interpolated into thrown errors or logs; error messages carry status code only |
| T-02-INJ | Injection (downstream) | transcript text | mitigate | Transcript persisted as plain column data only — never eval'd, shell-interpolated, or executed (consumed as data by Phase 3+) |
| T-02-PATH2 | Tampering | audioAbsPath | mitigate | Caller (enrichFinding, Plan 01) confines the path under MEDIA_DIR before passing; transcribe opens only the provided absolute path |
| T-02-SC | Tampering | npm installs | mitigate | No new dependency — uses built-in fetch + Bun.file; halt for legitimacy checkpoint if any install is proposed |
</threat_model>
