<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local keyframe files → Anthropic API | image bytes leave the host to a third-party vision service |
| env → outbound header | ANTHROPIC_API_KEY placed in the x-api-key header |
| Claude response → finding row | visual_summary / onscreen_text are attacker-influenceable (reel imagery + on-screen text); stored as data only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-COST | DoS (cost) | keyframe count + max_tokens | mitigate | Slice keyframes to MAX_FRAMES before encoding; set an explicit bounded max_tokens on the request (caps per-reel spend per the security model) |
| T-02-SEC3 | Information Disclosure | ANTHROPIC_API_KEY | mitigate | Key only in x-api-key header; never interpolated into thrown errors or logs; errors carry status code only |
| T-02-INJ2 | Injection (downstream) | onscreen_text / visual_summary | mitigate | Model output persisted as plain column data only — never eval'd, shell-interpolated, or executed (consumed as data by Phase 3+) |
| T-02-PATH3 | Tampering | keyframeAbsPaths | mitigate | Caller (enrichFinding, Plan 01) confines paths under MEDIA_DIR before passing; vision opens only the provided absolute paths |
| T-02-SC | Tampering | npm installs | mitigate | No new dependency — uses built-in fetch + Bun.file; halt for legitimacy checkpoint if any install is proposed |
</threat_model>
